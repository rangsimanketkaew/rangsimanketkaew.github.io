## Code Mania 1010

สวัสดีครับ บล็อกนี้ผมจะมาสรุปทอล์กที่ผมได้ไปพูดที่งาน Code Mania 1010 ครับ ซึ่งงาน Code Mania 1010 
เป็นงานที่รวมเหล่าโปรแกรมเมอร์ไทยมาแลกเปลี่ยนความรู้กัน ซึ่งในการจัดงานแต่ละรอบก็จะมี Theme ของงานที่ค่อนข้างชัดเจน โดยปีนี้ Theme ของงานคือ All About Passion และงานได้จัดขึ้นที่ 
True Digital Park (TDPK) กรุงเทพมหานคร เมื่อวันอาทิตย์ที่ 15 ธันวาคม พ.ศ. 2562

<br>

## ผมพูดเกี่ยวกับอะไร?

<img src="/assets/img/posts/code-mania-1.png" width="100%" height="auto" />

หัวข้อที่ผมได้ไปพูดในงานก็คือ "Low-Level Programming Language for High performance Scientific Computing"
แปลเป็นไทยง่าย ๆ ว่า การเขียนโปรแกรมคอมพิวเตอร์ระดับล่างสำหรับการคำนวณทางวิทยาศาสตร์ขั้นสูง เริ่มต้นผมเปิดประเด็นด้วยการยกตัวอย่างภาพยนตร์เรื่องต่าง ๆ ที่ล้วนแล้วแต่คณิตศาสตร์และวิทยาศาสตร์ทั้งสิ้น หลาย ๆ คนรู้จักภาพยนตร์ที่ผมยกมา เช่น Interstellar หรือ Breaking Bad แล้วผมก็ได้อธิบายโยงต่อไปถึงว่าการที่เราจะทำงานที่เกี่ยวข้องกับวิทยาศาสตร์ เราไม่จำเป็นจะต้องอยู่ในห้องแล็ปที่มีแต่สารเคมี แต่เราสามารถใช้การเขียนโปรแกรมมาสร้างซอฟต์แวร์ของเราเองเพื่อให้มันคำนวณนู่นนี้นั่นให้เราออกมาเป็นตัวเลข
ก่อนที่จะนำผลการคำนวณที่ได้ออกมานั้นส่งต่อไปยังนักวิทยาศาสตร์สายทดลองต่อไป 

<br>

<img src="/assets/img/posts/code-mania-6.png" width="100%" height="auto" />

แต่จริง ๆ แล้ว Scientific computing เนี่ยไม่ใช่ Computer science หรือ Software engineer นะครับ แตกต่างกันอย่างสิ้นเชิง เราไม่ได้เน้นไปที่การเข้าใจหรือการทำงานของ CPU หรือการสร้างอัลกิริทึมใหม่ ๆ แต่เราเน้นไปที่เราเขียนโปรแกรมขึ้นมาเพื่อคำนวณและตอบโจทย์ทางด้านวิทยาศาสตร์ เพราะฉะนั้นก็จะไม่แปลกถ้าหาก Code ที่เราเขียนนั้นมันรันได้ช้ามาก เพราะเราไม่ได้เรียนหรือเน้นมาทางด้าน Software Design โดยตรง นั่นหมายความว่า Scientific computing ก็ต้องพึ่งพาอีก 2 ฝั่งด้วยเช่นกัน

<br>

## อยากเห็นภาพมากกว่านี้

<img src="/assets/img/posts/code-mania-2.png" width="100%" height="auto" />

ตัวอย่างของ Scientific computing ก็จะมีเช่น การคำนวณทำนายหุ้น การทำนายการเกิดแผ่นดินไหวหรืออุทกภัย การศึกษาระดับอนุภาคหรือโมเลกุล การออกแบบและจำลองยาที่เข้าไปทำลายเชื้อโรคในเซลล์ของสิ่งมีชีวิต สามารถประยุกต์ไปใช้กับปัญญาประดิษฐ์ได้ด้วย ซึ่งมันมีแอพพลิเคชั่นเป็นร้อยเลยครับ เป็นศาสตร์ที่กว้างมาก ๆ สำหรับวิทยาศาสตร์เชิงคำนวณ

<br>

## แล้วมันเกี่ยวกับ HPC ยังไง

แต่การที่จำทำให้โปรแกรมเราคำนวณออกมาได้เร็ว ๆ นั้น เราต้องการคอมพิวเตอร์ที่มีประสิทธิภาพสูงครับ ไม่ใช่คอมพิวเตอร์หรือโน๊ตบุ้คทั่วไปที่เราใช้กัน แต่จะต้องเป็นซุปเปอร์คอมพิวเตอร์ (Supercomputer) ที่ถูกสร้างและออกแบบมาเพื่อ High Performance Computing (HPC) หรือการคำนวณขั้นสูงโดยใช้ CPU cores หลาย ๆ ร้อยหรือหลายพันตัวเพื่อมาช่วยกันรันงานที่เราส่งเข้าไป และแน่นอนว่ามันย่มอให้ผลลัพ์ของการคำนวณที่รวดเร็ว เร็วแบบเร็วมาก ๆ ซึ่งผู้ให้บริการ HPC ก็มีอยู่หลายเจ้า และแน่นอนว่าหลาย ๆ คนก็น่าจะรู้จักผู้ให้บริการ Cloud Computing เช่น AWS, Google Cloud Platform และ Microsoft Azure กันอย่างแน่นอน ซึ่งผู้ให้บริการเหล่านี้นี่เองที่เป็นผู้ให้เราสามารถนำงานที่ต้องการพลังานในการคำนวณแบบหนัก ๆ ส่งขึ้นไปรันได้ 

<br>

## ก็เลยลองยกตัวอย่าง ..

ถ้าเล่าให้ฟังอย่างก็คงจะไม่เห็นภาพ ผมก็เลยมีการแสดง Demo ให้ดู (ซึ่งไม่รู้ว่าทำให้คนงงมากกว่าเดิมหรือเปล่า)

### Demo 1

<img src="/assets/img/posts/code-mania-3.png" width="100%" height="auto" />

<br>

สมมติว่าผมอยากจะไปซื้อโทรศัพท์หรือโน๊ตบุ๊คใหม่สักเครื่องนึง แน่นอนว่าเราก็อาจจะมีรุ่นที่เราลังเลสัก 2-3 รุ่น เมื่อเราเดินไปที่ร้านขายอุปกรณ์ไอทีตามห้างใหญ่ ๆ ก็จะมีพนักงานขายรีบเข้ามาอธิบายสรรพคุณและเชียร์เราให้ซื้อรุ่นนี้หรือรุ่นนั้นทันที แล้วเราจะรู้ได้ไงว่ารุ่นไหนที่มันจะเร็วกว่ากัน? แน่นอนว่าทุกคนอยากได้โทรศัพท์หรือโน๊ตบุ๊คที่เร็วและมี CPU ที่แรง ๆ นั่นก็คือเราควรจะต้องมีการประเมินและเปรียบเทียบประสิทธิภาพ (Benchmarking) ของอุปกรณ์ชิ้นนั้นก่อนซื้อครับ!! ใช้แล่วครับ Demo นี้ผมสอนวิธีการเช็คความเร็วของโทรศัพท์แบบง่าย ๆ และได้ผลที่แม่นยำแม้แต่พนักงานขายยังต้องฟัง โดยใช้แอพที่ชื่อว่า Mobile Linpack (Download ได้ฟรีทั้ง iOS และ Android) ดาวโหลดแอพเสร็จ เปิดแอพ แล้วกดรันเลย แอพมันจะคำนวณคะแนนออกมาให้ พร้อมทั้งมี Ranking เปรียบเทียบความเร็วของมือถือเรากับซุปเปอร์คอมพิวเตอร์ตัวแรงที่ญี่ปุ่นให้อีกด้วย

### Demo 2

<img src="/assets/img/posts/code-mania-4.png" width="100%" height="auto" />

<br>

ยังไม่พอครับ ผมยังมีการยกตัวอย่างการวัดความเร็วหรือประสิทธิภาพของเครื่องซุปเปอร์คอทพิวเตอร์ให้ดูด้วย ถามว่ามันมีความสำคัญยังไง? ยกตัวอย่างง่าย ๆ เช่น เราไปใช้บริการ Cloud ของ Amazon นั่นก็คือ AWS EC2 แล้วกันนะครับ สมมติว่าเราเรียกใช้ Compute Node สัก 10 เครื่อง โอ้วโหววว ถ้าเปิดนาน ๆ นี่คงแพงน่าดู แล้วทีนี้เราจะรู้ได้ไงว่าเจ้า Compute Node ทั้ง 10 เครื่องนี้มันทำงานได้เร็วจริงตามที่ AWS เคลมไว้? แน่นอนว่าวิธีที่จำรู้ได้ก็คือการวัดควาเร็ว ซึ่งใน Demo นี้ผมก็แสดงวิธีการวัดคะแนนความเร็วออกมาโดยการใช้โปรแกรมที่ชื่อว่า Linpack ซึ่งมันเป็นซอฟต์แวร์ตัวเดียวกันกับที่ทาง TOP500 เขาใช้วัดความเร็วของซุปเปอร์คอมพิวเตอร์ทั่วโลกและตัดอันดับทุก ๆ ปี ปีละ 2 ครั้ง โดยตอนนี้ตัวที่เป็นเบอร์หนึ่งก็คือ SUMMIT Supercomputer ของอเมริกาครับ

## อิหยังวะ!

<img src="/assets/img/posts/code-mania-5.png" width="100%" height="auto" />

<br>

ระหว่างที่ผมพูดอยู่นั้นผมสังเกตเห็นได้เลยว่าหลาย ๆ คนงงแน่นอนครับว่าผมกำลังพูดเกี่ยวกับอะไร จริง ๆ แล้วสิ่งนึงที่ผมอยากสื่อออกไปก็คือกระบวนการหรือวิถีชีวิตของคนที่งานทางด้านนี้ ซึ่งเราก็มักจะวุ่นอยู่กับของสองอย่างคือ 1) Low-Level language เช่น C หรือ C++ 2) Supercomputer โดยเราเองนั้นไม่ได้สนใจ API หรือความสวยงามหรือหน้าตาของโปรแกรม เช่น UX/UI เลย เราสนใจเพียงตัวเลขที่คำนวณหรือทำนายออกมานั้นต้อง 1) ถูกต้องและแม่นยำ 2) คำนวณออกมาได้รวดเร็ว 

ท้ายที่สุดแล้วมันไม่สำคัญว่าภาษาที่เราจะใช้ในการเขียนนั้นมันจะ Low หรือจะ High ขอแค่เราสามารถเขียนเพื่อให้คอมพิวเตอร์หรือเครื่องสามารถทำงานให้เราได้ตามที่ต้องการก็พอแล้ว เพราะระดับของภาษาถูกกำหนดขึ้นมาโดยอ้างอิงจากการที่เรามนุษย์เป็นตัวเทียบ ภาษาที่เราใช้พูดคุยกันอยู่ทุกวันนี้จึงถือว่าอยู่สูงกว่าภาษาโปรแกรมมิ่งทั้งหมด ดังนั้นทุกภาษจึงเป็น Low-Level ทั้งหมดครับ

<br>

## สิ่งที่ผมอยากบอกก็คือ ...

ในจักรวาลของ Programming นั้นมันช่างกว้างใหญ่ไพศาลมากครับ ที่ผมกำลังจะบอกก็คือมันไม่ได้มีแค่ Web & App เท่านั้น แต่มันยังมีโลกอีกใบนึงที่เรียกว่า Scientific Computing หรือ Computational Science หรือการคำนวณเชิงวิทยาศาสตร์อยู่ด้วย ซึ่งสิ่งเหล่านี้มันอาจจะดูเหมือนว่ามันไกลตัวเรามาก อะไรก็ไม่รู้ซับซ้อนไปหมด สมการนู่นนี่นั่น ปวดหัวตาลาย ซึ่งแค่อยากมาแชร์ให้เพื่อน ๆ ได้ฟังกัน :)

<img src="/assets/img/posts/code-mania-7.png" width="100%" height="auto" />

<br>

สิ่งที่อยากบอกอีกคือ วิทยาศาสตร์อยู่รอบตัวเรา มันคือกระบวนการ หลักการและเหตุผล การตัดสินใจ การลองผิดลองถูกและที่สำคัญคือความท้าทาย การที่เรียนรู้วิทยาศาสตร์ได้อย่างมีความสุขและสนุกก็คือจะต้องมี Passion นั่นเอง :)

<br>

## สุดท้ายแล้ว 

สิ่งที่ผมต้องการจะสื่อออกไปก็คืออยากให้คนฟังทุก ๆ คนได้เรียนรู้และเข้าใจนักวิทยาศาสตร์สายคำนวณไปพร้อม ๆ กัน ในขณะเดียวกันผมเองก็ได้รับความรู้และแลกเปลี่ยนทัศนคติหรือมุมมองใหม่ ๆ กับเพื่อน ๆ (ที่เพิ่งไปรู้จักกันในงาน) หลาย ๆ คนเลย โดยส่วนตัวผมคิดว่าการที่มีการจัดงานประเภทนี้ถือว่าดีมาก ๆ ครับ เป็นกำลังใจให้ทางผู้จัดงานได้มีกิจกรรมดี ๆ แบบนี้ออกมาเรื่อย ๆ นะครับ Keep Fighting!
